Audio Filter Component Architecture

Filter data_in process

Where the .wav file data is sent is determined. If address is 0 then it does not go through either filter and is sent straight to the output. If address is 1 it is going to be going through one of the filters.

Filter coefficients process

The arrays for the high and low pass filters are filled with the corresponding coefficients. The process chooses which array of coefficients to use based on the filter select vector.

Shift register process

The shift register (delay_reg) is used to sequentially delay the input data, enabling the filter to process a sequence of input values (samples) in order. This delay mechanism is critical for implementing Finite Impulse Response (FIR) filters, where previous input samples are multiplied by pre-determined coefficients and then summed to produce the output.

Multiplier generator process

The multiplier takes the input signal and multiplies it with the corresponding coefficient. Since the input and output signals for this filter are 16 bits I used a for loop to instantiate 16 multipliers.
